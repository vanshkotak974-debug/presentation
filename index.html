Introduction to CSS Transitions

  • CSS Transitions allow smooth changes between property values over a specified duration
  • They enhance user experience by making interactions feel natural and polished
  • Transitions are triggered by state changes like hover, focus, or active states
  • Only animatable properties can be transitioned (color, size, position, opacity, etc.)
  • No JavaScript required â€” pure CSS solution for simple animations
CSS Code 
button {
  background: #4cc9f0;
  transition: background 1s;
}

button:hover {
  background: #f72585;
}

Why Use CSS Transitions?

  • Create smooth visual changes instead of abrupt jumps between states
  • Improve user experience by providing visual feedback for interactions
  • Guide user attention to important elements and interactions
  • Add polish and professionalism to your website with minimal code
  • Performance optimized - browsers can hardware accelerate transitions
CSS Code 
a {
  color: #212529;
  transition: color 0.5s;
}

a:hover {
  color: #f72585;
}

CSS Transition Syntax

  • Format: transition: property duration timing-function delay;
  • property: The CSS property to animate (or 'all' for all properties)
  • duration: How long the transition takes (in seconds or milliseconds)
  • timing-function: How the intermediate values are calculated
  • delay: How long to wait before starting the transition
CSS Code 
div {
  width: 100px;
  transition: width 2s ease-in-out 0.5s;
}

div:hover {
  width: 200px;
}

Hover to see the delayed width transition

5

Individual Transition Properties

  • Instead of shorthand, you can use individual transition properties
  • transition-property: Specifies which properties to transition
  • transition-duration: Defines how long the transition lasts
  • transition-timing-function: Controls the acceleration curve
  • transition-delay: Specifies when the transition will start
CSS Code 
.box {
  background: #ff9eaa;
  transition-property: background;
  transition-duration: 1s;
  transition-timing-function: ease;
  transition-delay: 0.3s;
}

.box:hover {
  background: #f72585;
}

Hover to see the delayed background transition

6

Timing Functions

  • Timing functions control the acceleration pattern of transitions
  • linear: Constant speed throughout the transition
  • ease: Slow start, fast middle, slow end (default)
  • ease-in: Slow start, fast finish
  • ease-out: Fast start, slow finish
  • ease-in-out: Slow at both beginning and end
CSS Code 
.box {
  width: 100px;
  transition: width 2s ease-in-out;
}

.box:hover {
  width: 200px;
}

Hover to see the ease-in-out timing function

7

Multiple Property Transitions

  • You can transition multiple properties simultaneously
  • Separate each property with a comma in the transition declaration
  • Each property can have its own duration, timing function, and delay
  • Use transition: all to transition all changing properties
  • Be careful with "all" as it can impact performance with many properties
CSS Code 
div {
  width: 100px;
  height: 100px;
  background: #52b788;
  transition: width 1s, height 1s, 
              background 1s, transform 1s;
}

div:hover {
  width: 150px;
  height: 150px;
  background: #2a9d8f;
  transform: rotate(10deg);
}

Practical Examples

  • Button hover effects for better user interaction feedback

9

Transition vs Animation

  • Transition: Property changes triggered by state changes
  • Animation: Runs automatically with @keyframes, can loop
  • Transitions are simpler and require less code for basic effects
  • Animations offer more control with multiple keyframes and iterations
  • Use transitions for simple state changes, animations for complex sequences
CSS Code 
/* Transition Example */
.box1 {
  background: #ff9e6d;
  transition: background 1s;
}
.box1:hover { 
  background: #f72585; 
}

/* Animation Example */
@keyframes colorChange {
  from { background: #ff9e6d; }
  to { background: #f72585; }
}
.box2 {
  animation: colorChange 2s infinite alternate;
}

Left: Hover transition | Right: Automatic animation
